# Reader

Локальный конвертер PDF/DOCX в Markdown для Windows.

Программа работает на компьютере пользователя и не отправляет документы в интернет. Подходит для папок, где вместе лежат обычные PDF, сканы PDF, смешанные PDF и DOCX.

## Как пользоваться

1. Откройте `Reader_Portable.exe`.
2. Выберите папку с документами.
3. Дождитесь завершения обработки.
4. Откройте новую папку `markdown_result_...`.
5. Главный результат находится в файле `00_ALL_DOCUMENTS.md`.

`Reader_Portable.exe` уже содержит Python, нужные библиотеки и Tesseract OCR. Ничего дополнительно устанавливать не нужно.

## Что создаётся

- `00_ALL_DOCUMENTS.md` - один общий Markdown-файл со всеми документами.
- `01_markdown` - отдельные Markdown-файлы по каждому документу, без технических служебных шапок.
- `02_problem_files` - появляется только если какой-то файл требует ручной проверки.

## Что обрабатывается

- PDF с обычным текстовым слоем.
- Отсканированные PDF.
- Смешанные PDF, где часть страниц с текстом, а часть является сканом.
- DOCX, включая текст и таблицы.

Старые Word-файлы `.doc` не обрабатываются. Их лучше пересохранить в `.docx` или PDF.

## Как работает OCR

В `Reader_Portable.exe` встроен Tesseract OCR с языками `rus`, `eng` и `osd`.

PDF обрабатываются постранично:

- если на странице нормальный текст, программа берёт его напрямую;
- если страница похожа на скан, запускается OCR;
- если встроенный текстовый слой похож на нечитаемый набор символов, программа тоже запускает OCR;
- почти пустые страницы отсекаются, чтобы OCR не превращал шум в случайный текст;
- несколько OCR-страниц обрабатываются параллельно.

## DOCX

DOCX читаются встроенным способом: программа открывает файл Word как стандартный DOCX-архив и извлекает текст из `word/document.xml`.

Это не временный обходной путь. Такой подход делает обработку DOCX устойчивее, потому что обычные Word-документы больше не зависят от внешних конвертеров. MarkItDown и Docling остаются запасными вариантами.

## Структура проекта

Корень проекта оставлен максимально простым:

- `README.md` - инструкция.
- `Reader_Portable.exe` - готовая локальная программа, если она собрана на компьютере.

Служебные файлы разложены по папкам:

- `src/reader/markdown_converter.py` - основной код программы.
- `config/requirements.txt` - версии Python-компонентов для запуска из исходников.
- `scripts/Запустить_конвертер.bat` - запуск из исходников.
- `scripts/Проверить_среду.bat` - проверка Python, библиотек и OCR.
- `scripts/Установить_Tesseract_OCR.bat` - установка OCR для запуска из исходников.
- `runtime/` - локальная техническая папка для Python-среды и OCR-данных при запуске из исходников. В GitHub не загружается.
- `.gitignore` - защита от случайной загрузки личных документов, результатов и больших сборочных файлов.

## Запуск из исходников

Обычному пользователю это не нужно. Используйте `Reader_Portable.exe`.

Если нужно запустить именно исходный код:

1. Откройте `scripts/Запустить_конвертер.bat`.
2. При первом запуске будет создана локальная папка `runtime/.venv`.
3. Компоненты будут установлены из `config/requirements.txt`.
4. После завершения откроется папка результата.

## Что не загружается в GitHub

В репозиторий намеренно не добавляются:

- исходные PDF/DOCX-документы;
- папки результатов `markdown_result_...`;
- локальная техническая папка `runtime`;
- готовые `.exe`;
- сборочные папки `build` и `dist`.

В GitHub хранится только исходный код, инструкция и служебные файлы проекта.

## Актуальность инструментов

Проверено 03.06.2026:

- `docling==2.97.0`.
- `markitdown[pdf,docx]==0.1.6`.
- `pymupdf==1.27.2.3`.
- Tesseract OCR в переносимой сборке: 5.4.0 с русским OCR.

Проверенные альтернативы:

- PaddleOCR может быть сильнее на сложных OCR-задачах, но заметно тяжелее для переносимого `.exe`.
- OCRmyPDF полезен для добавления OCR-слоя в PDF, но не решает задачу прямой конвертации PDF/DOCX в Markdown.
- NAPS2 удобен как отдельная программа для сканирования, но не подходит как минимальная основа этого конвертера.
