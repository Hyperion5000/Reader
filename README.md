# Reader

Локальный конвертер PDF/DOCX в Markdown для Windows.

Программа обрабатывает документы на компьютере пользователя и не отправляет исходные файлы в интернет. Она подходит для подготовки PDF и Word-документов к дальнейшему анализу: каждый документ преобразуется в отдельный Markdown-файл, а также создается общий файл со всеми документами.

## Что умеет

- Конвертирует PDF и DOCX в Markdown.
- Использует Docling, MarkItDown и PyMuPDF.
- Для бумажных PDF-сканов может использовать Tesseract OCR.
- Создает папку результата `markdown_result_...`.
- Формирует отчеты о конвертации и качестве распознавания.
- Не использует LLM/Ollama и не переписывает юридически важный текст нейросетью.

## Быстрый запуск на Windows

1. Скачайте проект.
2. Откройте файл `Запустить_конвертер.bat`.
3. При первом запуске программа сама создаст локальную папку `.venv` и установит Python-компоненты из `requirements.txt`.
4. Выберите папку с PDF/DOCX-файлами.
5. После завершения откройте созданную папку `markdown_result_...`.

## OCR для бумажных сканов

Если нужно распознавать PDF-сканы без текстового слоя:

1. Запустите `Установить_Tesseract_OCR.bat`.
2. Затем запустите `Проверить_среду.bat`.
3. В проверке должны быть видны языки OCR `eng` и `rus`.

Файлы OCR-языков и локальная установка Tesseract не хранятся в репозитории. Их можно установить через отдельный `.bat`-файл.

## Файлы проекта

- `markdown_converter.py` - основная программа.
- `requirements.txt` - закрепленные версии Python-компонентов.
- `Запустить_конвертер.bat` - обычный запуск программы.
- `Проверить_среду.bat` - проверка установленной среды.
- `Установить_Tesseract_OCR.bat` - установка OCR для сканов.
- `ИНСТРУКЦИЯ_конвертер.md` - подробная пользовательская инструкция.
- `АУДИТ_инструментов_2026-06-01.md` - проверка актуальности инструментов.

## Что не загружается в GitHub

В репозиторий намеренно не добавляются:

- исходные PDF/DOCX-документы;
- папки результатов `markdown_result_...`;
- локальная Python-среда `.venv`;
- кэш Python `__pycache__`;
- OCR-данные `tessdata`.

Это сделано, чтобы в GitHub попадала программа, а не личные документы и временные результаты.
