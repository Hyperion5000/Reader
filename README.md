# Reader

Локальный конвертер PDF/DOCX в Markdown для Windows.

Программа обрабатывает документы на компьютере пользователя и не отправляет исходные файлы в интернет. Подходит для папок, где вместе лежат обычные PDF, отсканированные PDF, DOCX и Word-документы, сохранённые в PDF.

## Самый простой запуск

Если у вас есть готовый `Reader_Portable.exe`, просто откройте его и выберите папку с документами. Python, Docling, MarkItDown, PyMuPDF и Tesseract OCR отдельно ставить не нужно.

Готовый `Reader_Portable.exe` не хранится в репозитории GitHub, потому что он большой. В репозитории лежит минимальный исходный код и инструкция.

## Запуск из исходного кода

1. Откройте `Запустить_конвертер.bat`.
2. При первом запуске программа создаст локальную папку `.venv` и установит компоненты из `requirements.txt`.
3. Выберите папку с PDF/DOCX-файлами.
4. После завершения откройте созданную папку `markdown_result_...`.

## Что создаётся после обработки

В результате остаётся минимум файлов:

- `00_ALL_DOCUMENTS.md` - один общий Markdown-файл со всеми документами.
- `01_markdown` - отдельные Markdown-файлы по каждому документу.
- `02_problem_files` - создаётся только если есть файл, который требует ручной проверки.

Отдельные технические отчёты больше не создаются.

## OCR для сканов

В переносимый `Reader_Portable.exe` встроен Tesseract OCR с языками `rus`, `eng` и `osd`.

PDF обрабатываются постранично. Если страница уже содержит нормальный текст, программа берёт текст напрямую. Если страница похожа на скан или встроенный текстовый слой выглядит как нечитаемый набор символов, программа улучшает изображение перед OCR и запускает Tesseract только для этой страницы. Почти пустые страницы отсекаются, чтобы OCR не превращал шум в случайный текст. Несколько OCR-страниц обрабатываются параллельно, поэтому большие сканы идут быстрее.

Если запускать программу из исходного кода и OCR не найден:

1. Запустите `Установить_Tesseract_OCR.bat`.
2. Затем запустите `Проверить_среду.bat`.
3. В проверке должны быть видны языки `rus` и `eng`.

## Что обрабатывается

- PDF с текстовым слоем.
- Отсканированные PDF.
- Смешанные PDF, где часть страниц уже с текстом, а часть является сканом.
- DOCX.

Старые Word-файлы `.doc` не обрабатываются. Их лучше пересохранить в `.docx` или PDF.

## Минимальный состав проекта

- `markdown_converter.py` - основная программа.
- `requirements.txt` - закреплённые версии Python-компонентов.
- `Запустить_конвертер.bat` - запуск из исходного кода.
- `Проверить_среду.bat` - проверка среды.
- `Установить_Tesseract_OCR.bat` - установка OCR для запуска из исходного кода.
- `.gitignore` - защита от случайной загрузки личных документов, результатов и больших сборочных файлов.

## Актуальность инструментов

Проверено 03.06.2026:

- `docling==2.97.0` - свежая версия на PyPI.
- `markitdown[pdf,docx]==0.1.6` - свежая версия на PyPI.
- `pymupdf==1.27.2.3` - свежая версия на PyPI.
- Tesseract OCR upstream имеет более новую ветку 5.5.x, но для переносимой Windows-сборки оставлен проверенный Tesseract 5.4.0 с русским OCR.

Проверенные альтернативы:

- PaddleOCR может быть сильнее на сложных многоязычных OCR-задачах, включая русский, но заметно тяжелее для переносимого `.exe`.
- OCRmyPDF полезен для добавления OCR-слоя в PDF, но он также использует Tesseract и не решает задачу прямой конвертации в Markdown.
- NAPS2 удобен как сканерная программа с OCR, но это отдельное приложение, а не минимальный пакет для PDF/DOCX в Markdown.

Для текущей цели выбран самый практичный набор: Docling + MarkItDown + PyMuPDF + Tesseract. Он остаётся локальным, понятным и переносимым.

## Что не загружается в GitHub

В репозиторий намеренно не добавляются:

- исходные PDF/DOCX-документы;
- папки результатов `markdown_result_...`;
- локальная Python-среда `.venv`;
- кэш Python `__pycache__`;
- OCR-данные `tessdata`;
- готовые `.exe` и сборочные папки.

Это сделано, чтобы в GitHub попадала только программа, а не личные документы и временные файлы.
