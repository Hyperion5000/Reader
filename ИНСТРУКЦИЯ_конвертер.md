# Локальный конвертер PDF/DOCX в Markdown

Этот инструмент переводит документы из PDF и Word в Markdown на вашем компьютере. Исходные файлы не меняются и никуда не отправляются.

В программе нет Ollama и нет встроенных нейросетевых редакторов. Конвертация работает через обычные локальные инструменты: Docling, MarkItDown, PyMuPDF и Tesseract OCR.

Проверенные версии компонентов закреплены в `requirements.txt`. Подробный аудит актуальности инструментов находится в файле `АУДИТ_инструментов_2026-06-01.md`.

## Как запустить

1. Откройте файл `Запустить_конвертер.bat`.
2. При первом запуске программа сама установит нужные Python-компоненты в папку `.venv`.
3. Выберите папку с документами.
4. Дождитесь завершения.
5. Откройте созданную папку `markdown_result_...`.

## Что будет внутри результата

- `00_ALL_DOCUMENTS.md` - один общий Markdown-файл со всеми документами.
- `01_raw_markdown` - текст сразу после извлечения.
- `02_clean_markdown` - основной вариант для работы: безопасно очищенный Markdown.
- `03_reports` - общий отчёт, отчёт качества и список проблем.
- `04_problem_files` - отдельные заметки по файлам, которые нужно проверить вручную.

## OCR для бумажных сканов

Для PDF, которые являются сканами реальной бумаги, нужен Tesseract OCR.

1. Запустите `Установить_Tesseract_OCR.bat`, если OCR ещё не установлен.
2. После установки запустите `Проверить_среду.bat`.
3. В проверке должны быть языки `eng` и `rus`.

Если Tesseract не установлен, PDF из Word и Word-файлы всё равно будут обрабатываться, но бумажные сканы могут дать слабый результат.

## Что брать для анализа

- Для одного документа берите файл из `02_clean_markdown`.
- Для всей папки берите `00_ALL_DOCUMENTS.md`.
- Если в отчёте качества есть предупреждения, сначала проверьте эти места по оригинальному PDF.

## Проблемные файлы

Если появятся файлы в `04_problem_files`, откройте `03_reports/problem_files.md`.

Обычно причины такие:

- скан слишком тёмный или кривой;
- OCR извлёк слишком мало текста;
- в PDF есть нечитаемые символы;
- файл повреждён или защищён.

Такие документы лучше проверить вручную по оригиналу, особенно даты, суммы, номера дел и ФИО.

## Проверка

Запустите `Проверить_среду.bat`, чтобы увидеть:

- установлен ли Docling;
- установлен ли MarkItDown;
- установлен ли PyMuPDF;
- найден ли Tesseract OCR;
- видит ли OCR языки `eng` и `rus`.
